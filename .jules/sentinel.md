2026-05-24-Excessive Data Exposure of Patient Phones-When retrieving all requests for nurses, the patient's phone number was unconditionally exposed as `patientPhone` even when the request was simply 'pending' or 'paid'. This contradicts the platform's rule of hiding patient/nurse phone numbers from each other until a request is confirmed. To solve this, the SQL query must conditionally select PII.
-Use conditional SQL directly in database queries (e.g., `CASE WHEN r.status = 'confirmed' THEN u.phone ELSE NULL END as patientPhone`) to mask Personally Identifiable Information before it is sent from the server.
